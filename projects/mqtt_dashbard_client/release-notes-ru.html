Исправления

Исправлена потеря условий скриптов после восстановления из бэкапа

При восстановлении резервной копии условия скриптов (сценариев) могли полностью пропадать. После восстановления все сценарии открывались пустыми, без условий.

  • Резервное копирование теперь корректно сохраняет все условия скриптов
  • При восстановлении условия восстанавливаются в полном объёме

Скрипты теперь можно редактировать в отключённом состоянии

Ранее открыть скрипт на редактирование можно было только если он включён. Это не позволяло изменить сценарий, который был выключен.

  • При открытии отключённого скрипта он остаётся выключенным — не включается автоматически
  • Все изменения сохраняются, скрипт остаётся выключенным
  • Чтобы включить скрипт после редактирования — используйте переключатель на главном экране сценариев

Предыдущая версия 1.19.0

Улучшения

УЛУЧШЕНИЕ

MQTT теперь работает через HiveMQ

Полностью переписан MQTT-клиент. Вместо устаревшего Eclipse Paho используется современный HiveMQ MQTT Client.

Что изменилось:
  • Более стабильное соединение — меньше обрывов
  • Улучшенная обработка ошибок и автоматическое переподключение
  • Корректное отображение статуса подключения
  • Поддержка всех протоколов: TCP, SSL, WebSocket, Secure WebSocket
Отложенные сообщения:
  • Если при отправке данных нет соединения — сообщение сохраняется в очередь
  • При восстановлении связи все отложенные сообщения автоматически отправляются
  • Сообщения старше 24 часов удаляются

Не требует действий от пользователя — улучшение работает автоматически после обновления.

УЛУЧШЕНИЕ

Улучшенный drag-and-drop виджетов

Переработан экран главной страницы с виджетами. Перетаскивать и организовывать виджеты стало удобнее.

Что изменилось:
  • Подсветка свободных ячеек — при перетаскивании видно, куда можно поставить виджет
  • Hover-подсветка — целевая позиция подсвечивается при наведении
  • Контекстное меню — долгое нажатие на виджет открывает меню действий
  • Улучшена логика перемещения между группами
  • Исправлено мерцание и сдвиги сетки при перетаскивании

ПРЕМИУМ

Новые возможности в скриптах

Скрипты (сценарии) стали ещё мощнее — добавлены новые типы условий и действий.

Новые возможности:
  • LogicalBlock — логический блок для группировки нескольких условий. Позволяет комбинировать условия через И / ИЛИ, создавая сложные сценарии:
    Пример: «Если температура > 30°C И влажность < 20%» —
    или: «Если кнопка нажата ИЛИ прошло больше часа»
  • StartAppCondition — новое условие, которое срабатывает при запуске приложения. Полезно для выполнения действий после перезапуска.
  • Уведомление звуком — как действие в сценарии можно воспроизвести звуковой сигнал или показать push-уведомление
Где найти:

Меню → Сценарии → создать или редактировать сценарий

В конструкторе условий появились новые блоки для добавления:

  • LogicalBlock (И/ИЛИ)
  • StartAppCondition
  • Действие: звуковое уведомление

ПРЕМИУМ

Загрузка сторонних SSL-сертификатов

Подключайтесь к брокерам с любыми SSL/TLS сертификатами, включая самоподписанные и корпоративные CA.

Самоподписанные сертификаты:
  1. При создании/редактировании брокера выберите протокол ssl:// или wss://
  2. Включите опцию Self-Signed Certificate
  3. Приложение будет доверять любому сертификату этого брокера
Подходит для тестовых и локальных брокеров (домашний сервер, Raspberry Pi).
Кастомные CA-сертификаты:
  1. Выберите протокол ssl:// или wss://
  2. Опция Self-Signed Certificate должна быть выключена
  3. Нажмите Select certificate file
  4. Выберите файл сертификата в формате .pem через системный файловый менеджер
  5. Имя выбранного файла отобразится на экране
Подходит для подключения к корпоративным или облачным MQTT-брокерам с собственным CA.
ПРЕМИУМ

Regex-парсинг данных

Новый способ извлечения данных из payload — регулярные выражения. Работает как дополнительный этап после JsonPath.

Трёхшаговый pipeline:
  1. JsonPath — извлечение поля из JSON (было и раньше)
  2. Regex (основной) — извлечение данных регулярным выражением
  3. Regex (вторичный) — постобработка результата
Пример: payload = {"temp": "temperature: 25.5C humidity: 60%"}
  JsonPath: $.temp → "temperature: 25.5C humidity: 60%"
  Regex: temperature:\s*([\d.]+)C → 25.5
Дополнительно:
  • Regex времени — отдельное выражение для извлечения временной метки из payload
  • Вторичный regex — применить ещё одно регулярное выражение к результату первого
Как настроить:
  1. Откройте редактирование виджета
  2. В разделе Topic In включите тумблер Regex parse data
  3. Заполните regex-паттерны для данных и времени
  4. При необходимости включите вторичный regex
